¿Qué es el Análisis Exploratorio de Datos (EDA)?

El EDA es la primera etapa para entender cualquier conjunto de datos. Imagina que tienes un enorme libro lleno de registros, y antes de hacer cualquier modelo o predicción, quieres explorar ese libro: saber qué columnas tiene, qué tipo de información hay, si hay datos faltantes, valores extraños o patrones visibles.

  • Filas: cada fila es un registro (ejemplo o transacción).
  • Columnas: cada columna es una variable (monto, fecha, tipo de transacción, si es fraude o no...)

El EDA te ayuda a responder preguntas como:

  • ¿Cuántos registros tenemos?
  • ¿Qué tipo de datos tiene cada columna (números, texto)?
  • ¿Existen valores faltantes o muy raros?
  • ¿Cómo se distribuyen las variables numéricas?
  • ¿Existen relaciones entre variables?

Estas respuestas permiten limpiar y preparar el dataset para modelos de detección de fraude.

¿Qué es pandas y por qué usarlo?

pandas es una librería de Python diseñada para trabajar con datos tabulares. Si alguna vez usaste Excel, pandas es algo similar pero mucho más poderoso y automático, perfecto para grandes volúmenes y análisis avanzados.

Con pandas podemos:

  • Leer datos de archivos CSV, Excel o bases de datos.
  • Visualizar las primeras filas para entender qué hay.
  • Filtrar filas y columnas según condiciones.
  • Limpiar datos faltantes o erróneos.
  • Calcular estadísticas básicas como medias, desviaciones, etc.

Ejemplo sencillo en Python con pandas:

import pandas as pd

df = pd.read_csv("datos.csv")
print(df.head())
print(df.describe())

¿Qué son las clases desbalanceadas y por qué importan en fraude?

En detección de fraude, la gran mayoría de transacciones son legítimas y solo una pequeña fracción son fraudulentas. Por ejemplo, 99% legítimas y 1% fraude. Esto crea un desequilibrio de clases.

Este desequilibrio es un problema porque:

  • Un modelo que predice siempre "no fraude" tendrá alta exactitud, pero será inútil para detectar fraudes.
  • Los algoritmos aprenden mejor cuando hay ejemplos equilibrados de cada clase.

Por eso usamos técnicas para balancear los datos.

Librería imbalanced-learn para balancear clases

En Python, la librería imbalanced-learn ofrece métodos fáciles para balancear clases, como:

  • RandomOverSampler: duplica ejemplos de la clase minoritaria.
  • SMOTE: crea ejemplos sintéticos basados en vecinos cercanos.

Ejemplo de código:

from imblearn.over_sampling import RandomOverSampler

sampler = RandomOverSampler(random_state=42)
X_resampled, y_resampled = sampler.fit_resample(X, y)

Documentación completa: imbalanced-learn

Técnicas para balancear clases

Las técnicas más comunes para balancear clases son:

  • Oversampling: crear o copiar más ejemplos de la clase minoritaria (fraude).
  • Undersampling: reducir la cantidad de ejemplos de la clase mayoritaria (legítimos).
  • SMOTE: método que crea ejemplos nuevos sintéticos para la clase minoritaria usando vecinos similares.

Cómo analizar el dataset

Para analizar un dataset de fraude financiero, sigue estos pasos:

  1. Revisión inicial: inspecciona las primeras filas, tipos de datos, valores faltantes con pandas (head(), info(), isnull().sum()).
  2. Estadísticas descriptivas: usa describe() para ver medias, desviaciones y detectar valores extremos.
  3. Detección de valores faltantes y outliers: decide si imputar, eliminar o marcar para revisión.
  4. Visualizaciones: histogramas para distribuciones, boxplots para outliers, heatmaps para correlaciones.
  5. Balanceo de clases: aplicar oversampling o SMOTE si la clase fraudulenta es minoritaria.
  6. Documentación: mantener notas claras y explicar cada paso para replicabilidad.

Tutorial paso a paso para obtener el producto final entregable

Este tutorial te guiará para crear un notebook interactivo donde realices un análisis exploratorio de datos (EDA) sobre un dataset público de fraude financiero, limpiar y anotar el dataset, y generar visualizaciones claras que expliquen tus hallazgos.

  1. Instalar el entorno de trabajo
    • Descarga e instala Anaconda, que incluye Python y Jupyter Notebook.
    • Alternativamente, puedes usar Google Colab, una plataforma gratuita en la nube sin instalaciones.
  2. Crear y preparar el notebook
    • Abre Jupyter Notebook o Google Colab y crea un nuevo notebook.
    • Instala las librerías necesarias ejecutando en una celda: 
      !pip install pandas matplotlib seaborn imbalanced-learn
    • Importa las librerías básicas: 
      import pandas as pd
  import matplotlib.pyplot as plt
  import seaborn as sns
  from imblearn.over_sampling import RandomOverSampler
  3. Cargar el dataset
    • Descarga el dataset público de fraude desde Kaggle.
    • Sube el archivo CSV a tu entorno y cárgalo en pandas: 
      df = pd.read_csv("creditcard.csv")
    • Explora las primeras filas para entender la estructura: 
      df.head()
  4. Exploración inicial
    • Revisa tipos y valores faltantes: 
      df.info()
  df.isnull().sum()
    • Obtén estadísticas descriptivas: 
      df.describe()
    • Observa la distribución de la variable objetivo ('Class'): 
      df['Class'].value_counts()
  5. Limpieza y anotación
    • Trata los valores faltantes (si hay): 
      # Ejemplo: imputar con mediana
  df.fillna(df.median(), inplace=True)
    • Detecta outliers con boxplots y rango intercuartílico (IQR): 
      Q1 = df.quantile(0.25)
  Q3 = df.quantile(0.75)
  IQR = Q3 - Q1
  outliers = ((df < (Q1 - 1.5 * IQR)) | (df > (Q3 + 1.5 * IQR))).sum()
    • Marca o elimina outliers según criterio.
    • Añade anotaciones si es necesario con una nueva columna explicativa.
  6. Balancear las clases
    • Separa características y etiquetas: 
      X = df.drop('Class', axis=1)
  y = df['Class']
    • Aplica oversampling para balancear: 
      ros = RandomOverSampler(random_state=42)
  X_res, y_res = ros.fit_resample(X, y)
  7. Visualizaciones básicas
    • Histograma de montos o variables clave: 
      sns.histplot(df['Amount'], bins=50)
  plt.show()
    • Boxplot para detectar outliers: 
      sns.boxplot(x='Class', y='Amount', data=df)
  plt.show()
    • Matriz de correlación: 
      corr = df.corr()
  sns.heatmap(corr, cmap='coolwarm', annot=False)
  plt.show()
  8. Documentar el análisis
    • Comenta cada bloque de código en el notebook explicando qué haces y por qué.
    • Resume hallazgos clave en celdas de texto para que sea claro y replicable.
  9. Guardar y compartir
    • Guarda el notebook con un nombre descriptivo, por ejemplo, eda_fraude_creditcard.ipynb.
    • Incluye el dataset limpio y anotado junto con el notebook para entregar.
    • Prepara un README breve con el resumen de tu trabajo y próximos pasos.
